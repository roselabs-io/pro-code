The Map Around Claude Code

A follow-up to the first piece — the one on the harness and its guides and graders. One word — “harness” — turned out to mean two different things. Pulling them apart is a simple way to lay out what sits around the tool a lot of us use.

TL;DR
  • Everything an agent involves sits at one of three heights: model · harness (runtime) · control-harness. Call them L0–L2.
  • Claude Code is a harness — the meaning the field's been settling on: the runtime that makes an agent run. It's transversal; it stripes through several layers at once.
  • The layer it doesn't fully bring is the top one — the control-harness, split into guides (the AGENTS.md standard) and graders (the quality checks — the tools standardize fast, the domain rubric doesn't).
  • The grader slot is filling fast, from the general end in — most of it someone's already published, to grab and compose. Only the domain-specific tail still tends to need building. The skill is knowing which is which.

The last piece landed on a word, and the word was doing double duty. I'd been using “harness” for the layer a developer wraps around a coding agent to control what it produces — the guidance, and the checks. The field, meanwhile, has been settling the bare word “harness” on something else: the runtime that makes an agent run — Anthropic's Claude Agent SDK, the write-ups calling 2026 “the year of the agent harness.” Both are harnesses in the old sense — things strapped on to control a powerful engine — but they do different jobs, so they need telling apart. The runtime is the harness. The layer on top is the control-harness. Once they're separated, the rest of the layout is easy enough to sketch — which is what this is, for anyone who lives in Claude Code and wants to see what's around it.

01the heights

Three layers, bottom to top.

Anything an agent involves sits at one of a few heights. The model at the bottom — raw intelligence. The runtime that makes it actually run — the loop that calls the model, runs a tool, feeds the result back, and repeats, plus memory, context, and the connectors (MCP). And on top, the layer that controls the output — the guidance an agent is given, and the checks run on what it produces. Model, runtime, control-harness. Call them L0 to L2. For a Claude-Code native, the middle one is already home — usually unnamed.

02the untangling

Claude Code is the harness.

The distinction from last time is simple. When the field says “harness” now, it means the runtime — the loop that drives the model through tools and back, the context it manages, the connectors it reaches through. That is Claude Code. Claude Code is a harness — the field's harness, the L1 kind.

But it doesn't stay in the L1 box. It's transversal: it wraps a model it doesn't own below, it is the runtime in the middle, and it ships a thin slice of the control-harness above — the review pass, the ability to check its own work. So “where does Claude Code sit” has no single answer. It's a vertical stripe through the stack. Which is exactly why it feels like it covers everything — because it nearly does, except for the top.

03the layer on top

The control-harness: guides, and graders.

The one layer Claude Code doesn't fully bring is the top one — the control-harness, supplied for a particular body of work. It has two halves:

Guides — feedforward: the conventions, the context, how-it's-done-here. The field standardized this as AGENTS.md — an open spec since 2025, read natively by Cursor, Codex, Copilot and the rest. Claude Code is the holdout: it still reads CLAUDE.md, so you point it at AGENTS.md with an import or a symlink. Either way, everyone has one.

Graders — feedback: the quality checks on what the agent produced. (Anthropic's own word — in Demystifying Evals, “logic that scores some aspect of the agent's performance,” in three kinds: code-based, model-based, and human.) Does it pass, is it correct, does it clear the bar for this kind of work. The tooling for this is standardizing fast — LangSmith, Braintrust, OpenAI Evals, emerging YAML eval specs. What stays specific is the rubric: what “good” means in a particular domain — the part no one else has published, because it's specific to the work. (It's also a different thing from safety guardrails — the field's real-time content and safety blocks — a separate, already-crowding layer.)

04the plugin ecosystem

What's actually out there.

In the Claude Code world, that split has concrete shapes. The guides are CLAUDE.md / AGENTS.md (project context), skills (packaged know-how), slash commands (canned workflows), output styles. The graders are hooks (deterministic checks that fire on an event — run the linter, block on fail), review subagents (an LLM reading the output), and external evals. Plugins and their marketplaces just bundle these together. And the library for each clusters the same way — thick at the agnostic, stack-level end, thinning toward the domain-specific.

And “out there” is itself a gradient. A plugin might be one person's GitHub repo, a pack in a third-party directory (claudemarketplaces.com, composio-community/awesome-claude-plugins), or something promoted into Anthropic's official catalog — with plenty of overlap and uneven quality in between. None of it is finished: community versions get promoted, forked, abandoned. So the usual move isn't adopting one whole — it's cherry-picking across them, composing the pieces that fit, and folding them into a marketplace a team runs itself.

And every one of these shapes is young. “Skills” is a 2025 word; plugins and their marketplaces came after; the list will read differently by the time this does. What's a skill today is a built-in tomorrow, what's a plugin becomes a default — so this map, like the ones that follow, is a still frame. Draw it in pencil.

05the map moves

The map is already moving.

A map like this is out of date the moment it's drawn — a still frame of something that won't hold still. Two currents keep redrawing it. The first is absorption: anything general and useful drifts downward, into the harness itself. Web search, memory, planning, a review pass — each started as something a developer bolted on, and each got pulled into the runtime: web search and web-fetch became native Anthropic tools in September 2025, a memory tool followed, and code review now ships as a native command (Claude Code's /code-review), with the deeper reviews still plugins. The general end of the control-harness keeps quietly emptying into the platform.

The second is how domain-specific the work is — how domain-y. Not all domains are equal. Web and SaaS work is so common it's barely a domain at all; call it domain-zero. Fintech and healthcare carry real constraints. Narrow verticals — industrial control, robotics, embedded — are sparse and walled. The more domain-y the work, the smaller the market, the fewer people building for it — for now. That line keeps sliding toward the specific: what a developer had to build by hand last year is suggested-by-default this year.

06where it lands

The grader slot is filling — from the general end in.

Put the two currents together and the last part of the map fills in. A developer works from Claude Code, and most of what the control-harness needs is already there to pick up. The grader slot is the clearest case: the quality checks for a given kind of work. It's filling fast, from the general end inward — occupied wherever a domain's idea of “good” is written down: web and SaaS conventions, and even big regulated fields, where Semgrep checks PCI and SOC 2 and Promptfoo ships FINRA and SEC plugins. It thins out as the work turns both more domain-y and more tacit — a niche vertical's safety rules, a team's own conventions, the isolation logic only this codebase needs — where “good” lives in a practitioner's head, and where a smaller market means fewer people have written anything down.

The filled cells are the useful part — there to lean on. Knowing someone's already published a finance rubric or a PCI check means grabbing and adapting it instead of writing one. The empty tail is the part a particular domain might still need built: checks too specific for anyone else to have written. It keeps shrinking as more gets written down and shared — each year, more of the map is something to reach for.

That's the map around Claude Code. The harness runs it (Anthropic's), the guides point it (AGENTS.md, everyone's), the graders check it (mostly out there already, and increasingly built-in). Knowing all this is for reading the landscape — leaning on what's already published, knowing it well enough to point others to it, and building the one corner a domain might still need.
